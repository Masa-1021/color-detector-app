# Circle Detector 要件定義書

## 1. 概要

### 1.1 アプリケーション名
Circle Detector（円形色検知アプリ）

### 1.2 目的
カメラ映像内の指定した円形領域の色を検知し、パトライトの状態を判定してMQTT経由でOracle DBにデータを送信する。

### 1.3 背景
工場設備のパトライト（信号灯）を監視し、設備稼働状況をリアルタイムで収集・記録するシステム。

## 3. 機能要件

### 3.1 モード

#### 3.1.1 編集モード
- 円領域の作成・編集・削除
- グループ（パトライト）の設定
- 色の登録
- マッピングルールの設定
- 設定の保存・読み込み

#### 3.1.2 実行モード
- リアルタイム色検出
- マッピングルールに基づく番号決定
- MQTT送信
- ステータス表示
- カメラ映像表示（ON/OFF選択可）

---

### 3.2 円領域

#### 3.2.1 作成方法
1. カメラ映像上でクリック → 中心点を設定
2. ドラッグしてカーソル移動 → 半径がリアルタイムで変化
3. マウスを離す → 半径が確定

#### 3.2.2 属性
| 属性 | 説明 |
|------|------|
| id | 自動採番（1, 2, 3...） |
| name | ユーザー定義の名前（任意） |
| center_x | 中心X座標 |
| center_y | 中心Y座標 |
| radius | 半径（ピクセル） |
| group_id | 所属グループ（パトライト）ID |
| colors | 登録された色リスト |

#### 3.2.3 編集・削除
- 円をクリック → 選択状態
- 選択中に編集ダイアログ表示
- 削除ボタンで削除

#### 3.2.4 制限
- 最大10個まで

---

### 3.3 グループ（パトライト）

#### 3.3.1 概要
複数の円を1つのパトライト（信号灯）としてグループ化する。

#### 3.3.2 属性
| 属性 | 説明 |
|------|------|
| id | 自動採番 |
| name | グループ名（例：パトライト1） |
| sta_no2 | ステーション番号2 |
| sta_no3 | ステーション番号3 |
| default_value | ルール不一致時のデフォルト送信番号 |
| circle_ids | 所属する円のIDリスト |

#### 3.3.3 送信単位
- 1グループ = 1番号を送信
- 複数グループがある場合、各グループごとに送信

---

### 3.4 色設定

#### 3.4.1 色の登録方法
1. **画面から取得**: カメラ映像の現在の色をスポイトで取得
2. **カラーパレット**: HSVまたはRGBで直接指定

#### 3.4.2 色の属性
| 属性 | 説明 |
|------|------|
| name | 色名（例：赤、緑、黄） |
| h_center | 色相（Hue）中心値 0-179 |
| h_range | 色相の許容範囲 |
| s_min / s_max | 彩度（Saturation）範囲 0-255 |
| v_min / v_max | 明度（Value）範囲 0-255 |

#### 3.4.3 1円に複数色
- 1つの円に複数の色を登録可能
- マルチカラーLED対応

---

### 3.5 点滅検出

#### 3.5.1 検出方法
- 既存の点滅検出ロジックを流用
- 一定時間内の色変化回数で判定

#### 3.5.2 パラメータ
| パラメータ | デフォルト値 | 説明 |
|------------|--------------|------|
| window_ms | 2000 | 判定ウィンドウ（ミリ秒） |
| min_changes | 3 | 最小変化回数 |
| min_interval_ms | 100 | 最小変化間隔 |
| max_interval_ms | 1500 | 最大変化間隔 |

#### 3.5.3 結果
- 点滅はフラグ（true/false）として扱う
- 「色 + 点滅中」の組み合わせでルール評価

---

### 3.6 マッピングルール

#### 3.6.1 ルール種別

**単一円ルール**
```json
{
  "type": "single",
  "conditions": [
    {"circle_id": 1, "color": "赤", "blinking": false}
  ],
  "value": 10
}
```

**単一円 + 点滅ルール**
```json
{
  "type": "single",
  "conditions": [
    {"circle_id": 3, "color": "緑", "blinking": true}
  ],
  "value": 20
}
```

**複合ルール（複数円の組み合わせ）**
```json
{
  "type": "composite",
  "conditions": [
    {"circle_id": 1, "color": "赤", "blinking": false},
    {"circle_id": 2, "color": "緑", "blinking": false}
  ],
  "value": 30
}
```

#### 3.6.2 評価順序
- 各ルールに優先度（priority）を設定
- 優先度が高い順に評価
- 最初にマッチしたルールの番号を送信

#### 3.6.3 デフォルト値
- どのルールにもマッチしない場合
- グループごとに設定されたデフォルト番号を送信

#### 3.6.4 設定方法
- GUI: 画面上でルールを追加・編集
- JSON: 設定ファイルを直接編集

---

### 3.7 送信設定

#### 3.7.1 ステーション番号
| 項目 | 設定方法 |
|------|----------|
| STA_NO1 | プルダウン選択（設定ファイルに固定リスト） |
| STA_NO2 | グループごとに設定 |
| STA_NO3 | グループごとに設定 |

#### 3.7.2 送信タイミング
| モード | 説明 |
|--------|------|
| 変化時のみ | 状態が変化したときだけ送信 |
| 定期送信 | 設定間隔（秒）で定期的に送信 |

設定で切り替え可能。

#### 3.7.3 送信データ形式
既存テーブル（HF1RCM01）と同じ形式：
| カラム | 説明 |
|--------|------|
| MK_DATE | タイムスタンプ (YYYYMMDDHHmmss) |
| STA_NO1 | ステーション番号1 |
| STA_NO2 | ステーション番号2 |
| STA_NO3 | ステーション番号3 |
| T1_STATUS | 送信番号 |

#### 3.7.4 通信障害時
- JSONLファイルに一時保管
- 復旧後に自動再送
- 既存の message_queue.py を流用

## 7. 外部インターフェース

### 7.1 MQTT
- 既存の mqtt_oracle_bridge.py を使用
- トピック: `equipment/status/{sta_no3}`
- ペイロード: JSON形式

### 7.2 Oracle DB
- 既存テーブル HF1RCM01 を使用
- カラム: MK_DATE, STA_NO1, STA_NO2, STA_NO3, T1_STATUS

---

## 8. 制約事項

1. 円の最大数: 10個
2. グループ数: 制限なし（実用上は円の数以下）
3. カメラ: USB接続またはRaspberry Pi Camera
4. ネットワーク: MQTT Broker (Mosquitto) が必要

---

## 9. 用語集

| 用語 | 説明 |
|------|------|
| 円（Circle） | カメラ映像上の監視領域。円形で指定 |
| グループ（Group） | 1つのパトライトに対応。複数の円を含む |
| パトライト | 信号灯。複数のランプ（色）を持つ |
| マッピングルール | 色の状態から送信番号を決定するルール |
| 点滅 | 一定時間内に色が変化する状態 |
